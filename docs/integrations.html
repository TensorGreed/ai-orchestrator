Security

Every inbound webhook is signature-verified

Triggers don't just listen on a path — they validate the vendor's signature scheme before the workflow runs, and they enforce replay windows where the vendor supports them.

Stripe

HMAC-SHA256 over {t}.{body} with a 5-minute replay window.

GitHub

HMAC-SHA256 over the raw body, matched in constant time against X-Hub-Signature-256.

Slack

Signed-secret verification with timestamp skew protection and a granular event filter.

Discord

Ed25519 signature verification using the application public key, with PING/PONG handshake.

Telegram

Constant-time match of X-Telegram-Bot-Api-Secret-Token against the configured secret.

Generic HMAC

Pluggable hmac_sha256 mode on any webhook_input node for custom vendors.
