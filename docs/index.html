Batch 5 Highlight

Cron automation that feels native to the orchestrator

`schedule_trigger` nodes let workflows run at precise intervals with timezone awareness. On startup, the scheduler registers active schedules, executes workflows with `triggerType: "cron"`, persists execution history, and reloads schedules whenever workflows change.

  • Supports expressions like `0 9 * * *` and `*/15 * * * *`
  • Only runs when the schedule node is a connected entry point
  • Dynamically updates jobs when workflows are saved, imported, or deleted

Example Schedules

0 9 * * * Daily morning digest
*/15 * * * * Quarter-hour sync and monitoring
0 0 * * 1 Weekly Monday report run
